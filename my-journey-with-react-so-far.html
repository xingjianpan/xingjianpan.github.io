My journey with React and React Native so far

Posted on Mon 27 March 2017 in javascript

第一回合: Javascript in 2013

2013年的时候曾经心血来潮的学了一阵JavaScript， 特意买了两本大书：

  • Javascript权威指南 第六版
  • Professional JavaScript for Web Developers

然后跟着how to learn javascript properly的在线教教程自学。

但是习惯了python简单明快的语法， 对于JavaScript总是觉得哪里不对。然后囫囵吞枣的看了一遍，算是对于JavaScript有了一个感性的认识。

接下来很自然的问题就是如何学以致用。因为之前有Django的经验，所以很自然的想到了JavaScript的web framework。一番搜索之后，找到了MEAN:

  • MongoDB
  • Express
  • Anuglar
  • Node.js

MEAN的教程很多，我比较推荐这本MEAN Web Development。
跟着教程实现了一个CRUD的网站，然后就放下了。当时的感想是： MEAN框架的最大优势是前后端使用同一个语言。 数据以JSON的方式传递，相当便利。

第二回合：JavaScript in 2016

一晃几年，基本没怎么碰过JavaScript。2017年1月，看到一篇文章How it feels to learn JavaScript in 2016。不禁惊讶于JavaScript世界(或者叫前端)翻天覆地的变化，正好春节有些时间，又一次的心血来潮想要学习JavaScript.

因为看了这篇文章，所以对于前端的复杂有些心理准备， 然后就是撸起袖子干了。

  • 打开ReactJs的官网，跟着Tutorial走了一下。不明觉厉，用component的方式构建网页的概念很独特，也很有吸引力。并且那个class的语法好像python，很熟悉，感觉不是在写JavaScript。
  • 如果要使用JavaScript最新的语法，需要安装一个叫Babel的东西。
  • 还要用webpack打包文件
  • 用npm管理各种依赖，这个见过
  • nodejs，这个也用过。
  • ...

既有新面孔，也有老朋友。在经历一番摸索之后，我发现自己终于可以用ES6的JavaScript写一段Hello Wolrd,打包成一个bundle.js文件, 然后在html.index中引用了。 虽然作者在how to learn javascript properly这篇文章中表达了对于前端过于复杂的不满，但是作为一个初学者，我觉得似乎并没有那么夸张。这其中最大的因素，可能是JavaScript长得越来越像Python了。

第三回合：Hello Redux

Redux是神器。如果说这次重拾JavaScript的最大收获，那就是接触了Redux的设计思想。 可以用一个公式描述Redux的逻辑： newState = currentState(action)

比如说一个Todo list应用：

  • curreneState: 当前的状态，比如显示状态(全部、未来一周、已完成)， 某个todo是否完成。
  • action: 某种行为，比如切换显示状态，或者标记项目为已经完成
  • newState: 对于当前状态施加某种行为后的新的状态。

一下子，APP内的世界不再那么乱糟糟了。你可以看到App历史上的每个行为以及相应的状态，甚至可以轻松的回到过去的某个时点。这种感觉，有点像是Neo突然看到了Matrix的本质。

难能可贵的是，Redux的作者Dan Abramov亲自制作了免费的视频教程:Getting started with Redux, 这也是目前最好的Redux教程了。

有了React, Redux两样利器，我终于有点精神想要做点什么了。一直以来我都觉得现在的新闻客户端都不好用，然后就用React搞了一个自己的新闻应用
